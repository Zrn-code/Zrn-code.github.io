# 程式流程控制

# 結構化程序設計

  • 目標
    設計出結構清晰、可讀性強、易於分工合作編寫的
  • 步驟
    • 自上而下的分析：把複雜的問題分解成若干小問題以後再解決
    • 模組化設計：將程序劃分為多個模組，每個模組獨立存放、完成一個特定功能。
    • 結構化編碼：使用基本控制結構控制程序的流程。

# 模組

  • 組成：可以是一條敘述句、一段程式、一個函式等等...
  • 基本特徵：僅有一個入口和一個出口
  • 模組間的關係：相互獨立、內聚性很強
