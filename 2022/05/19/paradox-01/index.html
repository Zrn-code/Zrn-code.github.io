# 著名的雞蛋掉落問題

我有雞蛋KKK 個，有一棟樓共有NNN 層，現在我從任意一層開始扔雞蛋，雞蛋可能會碎，可能不會；如果沒有碎，我可以撿起來繼續扔，現在想找到一個臨界樓層，從這個樓層以及該樓層以下往下扔，雞蛋都不會碎，從這個樓層以上往下扔，雞蛋都會碎。

# 基本思考

一顆雞蛋

如果只有一顆雞蛋就要測量出結果，就只能從第一層樓開始丟，每次增加一樓，直到雞蛋破裂為止。
因此，最少只需測量一次，最多則要測量到NNN 次才碎。

無限多顆雞蛋

如果現在有無限顆雞蛋，我們讓每個雞蛋都從中間樓層丟下，實踐二分搜尋法。
假設此時N=100N = 100N=100 的最差狀況，測量的次數只需要log2100=6.64<7log_2100 = 6.64 < 7log2​100=6.64<7 次即可。

兩顆雞蛋 - 基礎解

如果我們現在只有兩顆蛋，我們可以把它們想像成粗調節輪與細調節輪，

一顆雞蛋以 nnn 樓為單位搜尋，當該雞蛋破裂，就代表臨界樓層在該單位以下，
那我麼就從上一個單位開始一樓一樓的測試，測到雞蛋破裂為止。

假設此時的N=100N = 100N=100，其最差狀況測量次數需要100n+n−1\frac{100}{n}+n-1n100​+n−1 次。透過基礎的微分運算，我們知道−100n2+1=0\frac{-100}{n^2}+1 = 0n2−100​+1=0 時會有最小值。因此，在 n=10n = 10n=10 的時候，最差狀況也還是需要測量191919 次才能知道答案。

兩顆雞蛋 - 最優解

如果想讓最大次數最小化，也就要讓最小次數最大化，也就是說最大和最小之間的差最小。
假設第一枚雞蛋每多扔一次，第二枚雞蛋就少扔一次。假設扔第一個雞蛋從第 x 層開始扔，(最壞情況) 第二次間隔 x-1 層開始扔（也就是 x+x-1 層），第三次間隔 x-2 層開始扔，直到間隔 1 層開始扔。因此1+2+...+x−1+x≥1001+2+...+x-1+x \ge 1001+2+...+x−1+x≥100 , 解得 x≥13.6x \ge 13.6x≥13.6， 因此從141414 層開始扔，保證能找到臨界樓層，所需的次數最少。

# 解決思路

# 動態規劃 + 二分搜

定義dp[K][N]dp[K][N]dp[K][N] 為有KKK 個雞蛋，NNN 層樓情況下，找到臨界樓層至少要扔雞蛋的次數。
考慮狀態轉移方程，假設從 x 層開始扔第一個雞蛋，有兩種情況：

  1. 雞蛋碎了，說明臨界樓層在111~x−1x-1x−1 之間，且雞蛋碎了一個，因此情況轉變為dp[K−1][x−1]dp[K-1][x-1]dp[K−1][x−1]
  2. 雞蛋沒有碎，說明臨界樓層在xxx~NNN 之間，且雞蛋可以撿起來繼續扔，因此情況變為dp[K][N−x]dp[K][N-x]dp[K][N−x]

需要考慮最壞情況，因此得取這兩個情況的的最大值，再加上當前從 x 扔的這次，狀態轉移方程如下：
dp[K][N]=max(dp[K−1][x],dp[K][N−x])+1dp[K][N]=max(dp[K-1][x],dp[K][N-x])+1dp[K][N]=max(dp[K−1][x],dp[K][N−x])+1
但是第一枚雞蛋從xxx 層開始扔是我們假設的，第一枚雞蛋從哪一層開始扔會使得所需的次數最小呢？
這裡就需要我們進行枚舉了，1≤x≤N1 \le x \le N1≤x≤N, 因此對狀態轉移方程進行改進：

dp[K][N]=min1≤x≤N(max(dp[K−1][x−1],dp[K][N−x])+1)dp[K][N]=min_{1\le x \le N} (max (dp[K-1][x-1],dp[K][N-x])+1)dp[K][N]=min1≤x≤N​(max(dp[K−1][x−1],dp[K][N−x])+1)

此時，直接撰寫程式的話，dpdpdp 的複雜度是O(kn2)O(kn^2)O(kn2)，共有一個O(kn)O(kn)O(kn) 個狀態，對於每個狀態枚舉雞蛋的樓層xxx 需要O(n)O(n)O(n) 的時間。因此我們需要想辦法優化枚舉的時間複雜度。

我們觀察到dp(k,n)dp(k,n)dp(k,n) 是一個對於nnn 的單調函數，也就是說在雞蛋數kkk 固定的情況下，樓層數nnn 越多，需要的測量次數一定不會變少。在上述的轉移方程中，第一項T1(x)=dp(k−1,x−1)T_1(x) = dp(k-1,x-1)T1​(x)=dp(k−1,x−1) 是一個隨xxx 增加的單調遞增函數，第二項T2(x)=dp(k,n−x)T_2(x) = dp(k,n-x)T2​(x)=dp(k,n−x) 是一個隨xxx 增加而單調遞減的函數。

如果這兩個函數都是連續函數，那麼我們只需要找出這兩個函數的交點，在交點處就能保證這兩個函數的最大值最小。但在本題中，T1T_1T1​ 和T2T_2T2​ 都是離散函數。在這種情況下，我們需要找到最大的滿足T1(x)<T2(x)T_1(x) < T_2(x)T1​(x)<T2​(x) 的x0x_0x0​，以及最小的滿足T1(x)≥T2(x)T_1(x) \ge T_2(x)T1​(x)≥T2​(x) 的x1x_1x1​。

我們只需要比較在x0x_0x0​ 和x1x_1x1​ 處兩個函數的最大值，取一個最小的作為xxx 即可。在數學上，我們可以證明出 x0x_0x0​ 和x1x_1x1​ 相差 1，這也是比較顯然的，因為它們正好夾住了那個想像中的交點，並且相距盡可能地近。因此我們就可以使用二分搜的方法找出x0x_0x0​，再得到x1x_1x1​：

  • 我們在所有滿足條件的xxx 上進行二分搜。對於狀態(k,n)(k,n)(k,n) 而言，xxx 即為[1,n][1,n][1,n] 中的任一整數；

  • 在二分搜的過程中，假設當前這一步我們查找到了xmidx_{mid}xmid​，如果T1(xmid)>T2(xmid)T_1(x_{mid})>T_2(x_{mid})T1​(xmid​)>T2​(xmid​)，那麼真正的x0x_0x0​ 一定在xmidx_{mid}xmid​ 的左側，否則真正的 x0x_0x0​ 就會在 xmidx_{mid}xmid​ 的右側。

在得到了​x0x_0x0​ 後，我們可以知道x1x_1x1​​即為x0+1x_0+1x0​+1 ，此時我們只需要比較 max(T1(x0),T2(x0))max(T_1(x_0),T_2(x_0))max(T1​(x0​),T2​(x0​)) 和max(T1(x1),T2(x1))max(T_1(x_1),T_2(x_1))max(T1​(x1​),T2​(x1​))，取較小的那個對應的位置作為 xxx 即可。

這樣一來，對於給定的狀態(k,n)(k,n)(k,n)，我們只需要O(logn)O(log n)O(logn) 的時間，就可以找到最優的xxx，因此，總時間複雜度從O(kn2)O(kn^2)O(kn2) 就能降低至O(knlogn)O(knlog n)O(knlogn) 了。

# 單調性的處理

我們固定 kkk，隨著 nnn 的增加，對於狀態轉移方程中 dp(k−1,x−1)dp(k-1, x-1)dp(k−1,x−1) 這一項，它的值是不變的，因為它和 nnn 無關。
而對於狀態轉移方程中 dp(k,n−x)dp(k, n-x)dp(k,n−x) 這一項，隨著 nnn 的增加，它的值也會增加。從上面的方法中，我們知道 dp(k−1,x−1)dp(k-1,x-1)dp(k−1,x−1) 隨著 xxx 單調遞增，而 dp(k,n−x)dp(k, n-x)dp(k,n−x) 隨著 xxx 單調遞減，那麼當 nnn 增加時，dp(k,n−x)dp(k,n-x)dp(k,n−x) 對應的函數折線圖在每個整數點上都是增加的，因此在 dp(k−1,x−1)dp(k-1,x-1)dp(k−1,x−1) 不變的情況下，xoptx_{opt}xopt​ 是單調遞增的。

我們可以想像一條斜率為負的直線和一條斜率為正的直線，當斜率為負的直線（模擬 dp(k,n−x)dp(k, n-x)dp(k,n−x)）向上平移（模擬 nnn 的增加）時，它和斜率為正的直線（模擬 dp(k−1,x−1)dp(k-1, x-1)dp(k−1,x−1)）的交點會一直向右移動，這個交點就確定了xoptx_{opt}xopt​，這與方法一也是一致的。

因此當我們固定 kkk 時，隨著 nnn 的增加，dp(k,n)dp(k, n)dp(k,n) 對應的最優解的坐標 xoptx_{opt}xopt​ 單調遞增，這樣一來每個 dp(k,n)dp(k, n)dp(k,n) 的均攤時間複雜度為 O(1)O(1)O(1)。

# 數學的方法

反過來想這個問題：如果我們可以做 ttt 次操作，而且有 kkk 個雞蛋，那麼我們能找到答案的最高的 n 是多少？我們設 f(t,k)f(t,k)f(t,k) 為在上述條件下的 nnn。如果我們求出了所有的f(t,k)f(t,k)f(t,k)，那麼只需要找出最小的滿足 f(t,k)≥nf(t, k) \ge nf(t,k)≥n 的 ttt。

我們需要找出最高的 nnn，因此我們不必思考到底在哪裡扔這個雞蛋，我們只需要扔出一個雞蛋，看看到底發生了什麼：

  • 如果雞蛋沒有碎，那麼對應的是f(t−1,k)f(t-1,k)f(t−1,k)，也就是說在這一層的上方可以有f(t−1,k)f(t-1,k)f(t−1,k) 層；
  • 如果雞蛋碎了，那麼對應的是f(t−1,k−1)f(t-1,k-1)f(t−1,k−1)，也就是說在這一層的下方可以有 f(t−1,k−1)f(t-1,k-1)f(t−1,k−1) 層。

因此我們就可以寫出狀態轉移方程：
f(t,k)=1+f(t−1,k−1)+f(t−1,k)f(t, k) = 1 + f(t-1, k-1) + f(t-1, k)f(t,k)=1+f(t−1,k−1)+f(t−1,k)

邊界條件為：

  • 當 t≥1t \ge 1t≥1 的時候 f(t,1)=tf(t, 1) = tf(t,1)=t。
  • 當 k≥1k \ge 1k≥1 時，f(1,k)=1f(1, k) = 1f(1,k)=1。
