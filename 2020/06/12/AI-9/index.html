# 圖片分割

# 什麼是圖片分割

到目前為止，您已經看到了圖像分類，其中網絡的任務是為輸入圖像分配標籤或類。但是，假設您想知道對像在圖像中的位置，該對象的形狀，哪個像素屬於哪個對像等，在這種情況下，您將需要對圖像進行分段，即圖像的每個像素都是給一個標籤。因此，圖像分割的任務是訓練一個神經網絡，以輸出圖像的逐像素蒙版。這有助於以低得多的級別（即像素級別）理解圖像。圖像分割在醫學成像，自動駕駛汽車和衛星成像等領域有許多應用。
數據集由圖像，其相應的標籤和逐像素蒙版組成。遮罩基本上是每個像素的標籤。
每個像素都具有以下三種類別之一：
第 1 類：屬於寵物的像素。
第 2 類：像素與寵物接壤。
第 3 類：以上均不存在。

# 定義模型

此處使用的模型是經過修改的 U-Net。一個 U-Net 由一個編碼器（下採樣器）和解碼器（上採樣器）組成。為了學習魯棒的功能並減少可訓練參數的數量，可以將預訓練模型用作編碼器。
因此，用於此任務的編碼器將是預訓練的 MobileNetV2 模型，將使用其中間輸出，而解碼器將是 [Pix2pix 教程] 的 TensorFlow 示例中已經實現的上採樣塊。

輸出三個通道的原因是因為每個像素有三個可能的標籤。
可以將其視為多分類，其中每個像素都分為三類。

如前所述，編碼器將是經過預訓練的 MobileNetV2 模型，該模型已經準備就緒，可以在 tf.keras.applications 中使用。
編碼器由模型中間層的特定輸出組成，在訓練過程中不會對編碼器進行訓練。

解碼器 / 上採樣器只是在 TensorFlow 示例中實現的一系列上採樣塊。

# 訓練模型

現在，剩下要做的就是編譯和訓練模型。
這裡使用的損失函數 loss.SparseCategoricalCrossentropy（from_logits = True）。
使用此損失函數的原因是因為網絡正嘗試為每個像素分配一個標籤，就像多類別預測一樣。在真正的分割蒙版中，每個像素都有一個 {0,1,2}。這裡的網絡正在輸出三個通道。
本質上，每個渠道都在嘗試學習預測類別和損失。對於這種情況使用 SparseCategoricalCrossentropy（from_logits = True）作為損失。使用網絡的輸出，分配給像素的標籤是具有最高值的通道。
這就是 create_mask 函數正在執行的操作。
