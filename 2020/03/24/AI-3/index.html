# Tensorflow 架構圖

# 處理器

  • CPU: 每一台電腦一定有中央處理器 (CPU)，可以執行 TensorFlow
  • GPU: 圖形處理器 (顯卡)，GPU 可以達到數千個小型且高效率的核心，可以發揮平行運算的強大功能
  • TPU: TPU 是 google 為人工智慧所研發的專屬晶片，比 GPU 有更佳的執行性能

# 平台

  • Windows: 個人電腦最常用的作業系統
  • Linux: TensorFlow 可以在各種版本的 Linux 系統上使用
  • iOS: TensorFlow 可以在 iOS 行動裝置或 Mac OS 上執行
  • Android: 在 Android 上運行 TensorFlow，讓深度學習進入行動端，Android 裝置的執行效能也開始提升，更適合 TensorFlow 的運行
  • Raspbarry Pi: 樹莓派可以開發物聯網或機器人，提供人工智慧的功能
  • 雲端執行：可以透過雲端的大量的伺服器，加速學習模型的訓練

# Tensorflow Distributed Execution Engine

在深度學習中，最花時間的是模型上的訓練，尤其是大資料的訓練，TensorFlow 具備分散式運算能力，可同時在數百台的機器上執行訓練模型，大幅減少訓練時間

# 前端程式語言

TensorFlow 可由多種前端語言編寫，例如: Python、C++，但 Python 的支援最佳，又具備簡明、易學習、高生產力，物件導向、函數式的動態語言，應用非常廣泛。

# 高階 API

TensorFlow 本身是較低階的 API，所以設計模型時，需要自行設計，如：張量乘積、卷積等底層操作，好處是可以自訂義自己想要的各種模型，缺點是需要大量時間開發。
開發社群以 TensorFlow 為基底，開發出了 Keras、TF-Learn、TF-Slim 等等。讓開發者使用起來更加的舒適、簡潔。

# TensorFlow 簡介

# Tensor 張量

張量，是一種幾何實體，或是廣義上的數量，包含 純量、向量或矩陣
0 維的張量為純量，1 維的張量為向量，2 維以上的為矩陣

# Flow 資料流程

為了讓不同的程式語言共通使用，所有的 TensorFlow 都要先建立起 計算圖 ，這是張量運算及資料處理流程。

# TensorFlow 程式設計模式

TensorFlow 的核心是 計算圖，可以分成 建立 與 執行

# 建立計算圖

透過 TensorFlow 提供的模組，設計張量的運算流程，並建構起各種深度學習所需的模型

# 執行計算圖

建立好計算圖後，我們可以建立起 Session 執行計算圖。Session 的作用是在用戶端及執行裝置間做連結。這個連結可以將計算圖在不同裝置中執行，後續的資料傳遞，都必須 Session 取得執行結果。

# keras 介紹

# 優點及特色

  1. Keras 可以用最少的程式碼，就建立起深度學習的模型、訓練以及預測。
  2. 已內建各式樣的神經網路，例如: CNN、RNN
  3. 透過的後端引擎可在 CPU、GPU 上使用
  4. 淺顯易懂、可讀性、可維護性高，更具生產力

# 運作方式

Keras 是一個 model-level 的深度學習程式庫，因此只處理模型的建立、訓練及預測。
Keras 必須配合使用後端引擎進行運算，目前提供兩種後端引擎做使用: TensorFlow、Theano。

# Keras 程式設計模式

模型就像是疊蛋糕一樣，可依據想要的內容，自訂義各層不同的神經元
流程如下:

  1. 建立線性堆疊
  2. 加入輸入層、隱藏層
  3. 加入輸出層
  4. 訓練、預測
    十分的簡單輕鬆
